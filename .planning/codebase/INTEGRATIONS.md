# INTEGRATIONS

External services, models, CLI tools, and file formats this skill produces or consumes.

## Image generation API

**Single provider, single model.** All three image-generation scripts call the same endpoint:

- **OpenRouter** — `https://openrouter.ai/api/v1/chat/completions`
- **Model**: `google/gemini-3-pro-image-preview`
- **Auth**: `OPENROUTER_API_KEY` env var (Bearer token)
- **Transport**: raw `urllib.request` (stdlib) — no `openai` / `anthropic` SDK
- **Response**: base64-decoded inline image data
- **Call pattern**: chat-completions with multimodal `content` array — text prompt plus optional reference images (also base64-encoded) for likeness anchoring

Files:
- `skill/scripts/generate_front_cover.py` (lines 35–36, 60–62, 106)
- `skill/scripts/generate_back_cover.py` (lines 35–36, 60–62, 106)
- `skill/scripts/generate_chapter_images.py` (lines 43–45, 58, 78, 156–174) — supports protagonist `canonical_ref` / `strong_refs` / `custom_refs` from launch config for character consistency across chapters

`CONTRIBUTING.md` calls out FLUX / Imagen / Stable Diffusion / DALL-E adapter PRs as wanted — confirming Gemini-via-OpenRouter is the only integration today.

No LLM-text API calls are made from scripts. Editorial review and prompt generation happen inside Claude Code (the skill runtime) using `skill/templates/editorial-prompt.md.template`.

## EPUB / PDF tooling

EPUB and the master PDF are built by an **external Node CLI** (`kdp-book-generator`, separate npm package, not vendored in this repo). This skill's Python scripts wrap and post-process its output:

- `skill/scripts/build_book_md.py` — preprocesses manuscript markdown (inserts chapter image refs, promotes `## CHAPTER N` to `# CHAPTER N` for page-break pagination) and writes `_build_book.md` for `kdp-book-generator` to consume.
- `skill/scripts/epub_embed_images.py` — uses `pypdf` plus `zipfile` (EPUB is a zip) to splice generated chapter images into the EPUB after `kdp-book-generator` runs.
- `skill/scripts/build_print_pdf.py` — independent print-PDF path that bypasses `kdp-book-generator`:
  1. `pandoc` (subprocess) converts markdown to HTML body.
  2. Inline print CSS wraps it (6x9 trim, KDP margins, EB Garamond stack, `@page` directives).
  3. Headless Chrome / Chromium (`--headless --print-to-pdf`) renders to PDF.
- `skill/scripts/postprocess.py` — uses `pypdf` `PdfReader` / `PdfWriter` to stamp PDF metadata (`/Producer: kdp-book-generator + kdp-book-launch`, title, author).
- `skill/scripts/compose_cover_wrap.py` — uses Pillow to composite front + spine + back into a full KDP wrap with calculated spine width.
- `skill/scripts/build_visual_companion.py` — Pillow-based PDF builder for the lead-magnet "Visual Companion" PDF (cover page, per-chapter image pages with captions, end page).
- `skill/scripts/build_social_pack.py` — Pillow generates platform-sized graphics (Instagram square / story, Twitter, Facebook, etc.) from cover + chapter images.
- `skill/scripts/make_bw_variants.py` — Pillow converts color chapter images to grayscale for paperback/print interior.

No use of `reportlab` or `ebooklib`. All PDF assembly is either Pillow (`Image.save(..., 'PDF')`) or pypdf merge/metadata.

## KDP output targets

The pipeline produces every file Amazon KDP expects:

- **EPUB** — for Kindle ebook upload (built by `kdp-book-generator`, post-processed by `epub_embed_images.py`)
- **Paperback interior PDF** — 6x9 trim, B&W interior (`build_print_pdf.py` + `make_bw_variants.py`)
- **Hardcover interior PDF** — same pipeline, hardcover trim variant
- **Full cover wrap PDF/PNG** — front + spine + back, generated by `compose_cover_wrap.py`
- **Visual Companion PDF** — newsletter/subscriber bonus, by `build_visual_companion.py`
- **Social media pack** — multi-platform image pack via `build_social_pack.py`
- **KDP listing copy + A+ Content + press release** — text deliverables from `skill/templates/kdp-listing.md.template`, authored by Claude during Phase 5 of the runbook (`skill/SKILL.md`).

KDP-specific spec details (trim sizes, bleed, gutter, allowed file formats) live in `skill/references/kdp-specifications.md`.
