Remove committed `__pycache__/` artifacts and ensure `.gitignore` prevents them from being re-committed.

Output: `__pycache__/` removed from git tracking; `.gitignore` updated.
</objective>

  <files>(read-only)</files>
  <action>Run:
- `ls .gitignore` — confirm it exists
- `cat .gitignore` — capture current contents
- `find . -name __pycache__ -type d -not -path './.git/*'` — list all __pycache__ directories
- `git ls-files | grep __pycache__` — list any TRACKED __pycache__ files
Record which patterns are already present (`__pycache__/`, `*.pyc`, `__pycache__`, etc) and which __pycache__ dirs need removing from git.</action>

  <files>(git index)</files>
  <action>For each tracked __pycache__ path found in Task 1, run `git rm -r --cached <path>`. The `--cached` flag removes from the index only — files stay on disk (and will be re-generated by Python anyway). If Task 1 found NO tracked __pycache__ files, skip this task with a note.</action>

  <acceptance_criteria>

  </acceptance_criteria>
  <done>__pycache__ untracked from git.</done>

<task type="auto">
  <name>Task 3: Update .gitignore</name>
  <files>.gitignore</files>
  <read_first>.gitignore</read_first>
  <action>Ensure .gitignore contains both `__pycache__/` and `*.pyc` patterns. Only ADD missing patterns — do not duplicate existing ones. If both patterns are already present, this task is a no-op (note that and skip the edit).

Idempotent shell logic:
```
grep -qxF '__pycache__/' .gitignore || echo '__pycache__/' >> .gitignore
grep -qxF '*.pyc' .gitignore || echo '*.pyc' >> .gitignore
```

If the file ends without a trailing newline, normalize that first.</action>
  <verify>grep -qxF '__pycache__/' .gitignore && grep -qxF '*.pyc' .gitignore</verify>
  <acceptance_criteria>
    - .gitignore contains an exact-match line `__pycache__/`
    - .gitignore contains an exact-match line `*.pyc`
    - No duplicate lines were added
  </acceptance_criteria>
  <done>.gitignore covers Python bytecode.</done>
</task>
